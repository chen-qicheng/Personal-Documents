## 项目概述

本项目是一个**嵌入式硬件项目**，目标是开发一款"新品卷烟推广语音播报器"。这是一个有偿委托的课题任务，需在五月底前完成一个功能完整的样品，用于汇报展示，无需量产。

### 核心功能
- **红外感应触发**：消费者靠近柜台时自动激活设备
- **语音播报**：支持U盘导入TXT文本（TTS合成）或MP3/WAV音频文件
- **低功耗管理**：无人时自动休眠，延长电池续航
- **外观**：3D打印塑料外壳，精细美观

### 硬件架构
| 模块 | 选型 | 说明 |
|------|------|------|
| 核心控制 | STM32 或 ESP32 | 信号处理、逻辑控制、电源管理 |
| 感应模块 | HC-SR501 | 人体红外感应，距离1-3米可调 |
| 音频播放 | VS1053/PAM8403 + 3W扬声器 | 支持MP3/WAV解码 |
| 供电模块 | 18650锂电池 + Type-C充电 | 支持充电/电池双供电 |
| 外壳结构 | 光敏树脂3D打印 | 表面抛光打磨 |

### 软件需求
1. **感应与控制逻辑**：人体靠近启动播放，远离停止并休眠
2. **音频管理**：读取U盘TXT（TTS合成）或直接播放音频文件
3. **系统设置**：按键/串口调整灵敏度、音量、播放次数
4. **低功耗管理**：休眠唤醒机制，仅保留红外感应模块运行

## 技术栈

### 硬件
- **主控**：STM32 / ESP32
- **传感器**：HC-SR501 PIR人体红外感应
- **音频**：VS1053 解码芯片 / PAM8403 功放
- **电源**：18650锂电池、Type-C充电电路
- **结构**：3D打印（光敏树脂）

### 软件
- **嵌入式开发**：C/C++（Arduino框架或HAL库）
- **TTS服务**：调用免费语音合成API
- **文件系统**：支持FAT32格式U盘读取

## 开发规范与指导

### 代码重构原则（来自 REFACTORING.md）
1. **渐进式改进**：重构不是推倒重来
2. **优化设计**：重构本质是优化原有设计
3. **小步快跑**：随时随地消灭代码坏味道
4. **测试保障**：重构需有充分的UT/FT保驾护航
5. **人人皆可**：从修改变量名到调整架构都算重构
6. **功能分离**：改功能时不重构，重构时不改功能

### Git 使用规范
- **分支管理**：使用 `git switch` 进行分支切换/创建
- **提交规范**：使用 `git add -p` 进行拆分提交
- **撤销修改**：优先使用 `git restore` 而非 `git checkout`
- **标签管理**：版本发布时打标签并推送到远程
