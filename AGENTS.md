## 文档规范

### 文件命名
- 使用**中文**命名，语义清晰
- 避免特殊字符，使用 `-` 连接（如 `近视手术.md`）
- 分类编号采用 `01-`、`02-` 等前缀，便于排序

### Markdown 格式
- 使用 `#`、`##`、`###` 等层级标题组织内容
- 善用表格、列表、引用块增强可读性
- 适当添加 emoji 图标提升视觉识别度

### Git 使用
虽然这是个人知识库，但仍遵循规范的 Git 工作流：

**提交类型**：
| 类型 | 用途 | 示例 |
|------|------|------|
| `docs` | 文档更新、内容补充 | `docs: 完善跑步训练计划` |
| `feat` | 新增功能/内容 | `feat: 添加游泳训练笔记` |
| `refactor` | 重构整理 | `refactor: 重新组织文件夹结构` |
| `fix` | 修正错误 | `fix: 更正近视手术术语` |

## 使用建议

1. **定期回顾**：建议每周浏览一次各分类，更新进度
2. **及时归档**：临时文件（temp）定期清理，有价值的内容移入对应分类
3. **持续链接**：相关文档之间互相引用，形成知识网络
4. **行动导向**：不仅记录知识，更要记录行动和结果
