## 3. 硬件设计概要

### 3.1 核心控制模块

- **MCU选型**: STM32F103C8T6（72MHz，64KB Flash）
- **供电**: 3.3V稳压（HT7333 LDO）
- **主要接口**: UART（DFPlayer通信）、GPIO（红外输入、BUSY检测）

### 3.2 红外感应模块

- **模块型号**: HC-SR501
- **工作电压**: DC 5V
- **感应范围**: <120°锥角，距离3米内可调
- **触发方式**: 可重复触发模式（H模式）
- **输出信号**: 高电平3.3V（接STM32 PB0）

### 3.3 音频播放模块

- **模块型号**: DFPlayer Mini
- **通信接口**: UART，9600bps
- **存储介质**: U盘（FAT32格式，最大32GB）
- **音频格式**: MP3、WAV
- **功放输出**: 3W，直接驱动4Ω扬声器
- **控制引脚**: BUSY（低电平=播放中，接STM32 PB12）

### 3.4 供电模块

- **电池**: 18650锂电池单节（3.7V，2000mAh）
- **充电管理**: TP4056模块，Type-C接口
- **升压输出**: 5V/2A（供DFPlayer和传感器）
- **电池保护**: DW01A+FS8205A（过充/过放/过流保护）
- **续航估算**: 连续播放约13小时，待机约400小时

### 3.5 主要接口定义

| STM32引脚 | 功能 | 连接模块 |
|:---------:|------|----------|
| PA9 | USART1_TX | DFPlayer RX |
| PA10 | USART1_RX | DFPlayer TX（可选） |
| PB0 | GPIO输入 | HC-SR501 OUT |
| PB12 | GPIO输入 | DFPlayer BUSY |
| PA0 | ADC | 电池电压检测 |
| PA15 | GPIO输入 | 配置按键 |

## 4. 软件设计概要

### 4.1 软件架构

```
┌─────────────────────────────────────────────────────────────┐
│                      应用层 (Application)                     │
│         主控制逻辑 │ 播放管理 │ 配置管理                        │
├─────────────────────────────────────────────────────────────┤
│                       服务层 (Services)                       │
│    红外服务 │ DFPlayer驱动 │ 电源管理 │ 调试日志                │
├─────────────────────────────────────────────────────────────┤
│                       驱动层 (Drivers)                        │
│      GPIO │ UART │ ADC │ 看门狗 │ 时钟/电源                    │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 系统状态机

```
         上电初始化              检测到人体              播放完成
    ┌──────────┐           ┌──────────┐           ┌──────────┐
    │          │ ────────> │          │ ────────> │          │
    │ POWER_ON │           │ STANDBY  │           │ PLAYING  │
    │ 上电启动  │           │ 待机检测  │ <──────── │ 播放中   │
    │          │ <──────── │          │  人离开   │          │
    └──────────┘ 初始化完成 └────┬─────┘           └──────────┘
                                 │
                                 │ 长时间无活动/低电量
                                 v
                          ┌──────────┐
                          │          │
                          │  SLEEP   │
                          │ 深度休眠  │
                          │          │
                          └────┬─────┘
                               │ 红外中断唤醒
                               └────────────────>
```

**状态说明**:
- **POWER_ON**: 上电初始化，完成各模块启动
- **STANDBY**: 待机检测，红外模块工作，检测人体靠近
- **PLAYING**: 播放中，DFPlayer播放音频
- **SLEEP**: 深度休眠，低功耗模式，红外中断唤醒

### 4.3 核心功能模块

| 模块 | 功能说明 | 关键特性 |
|------|----------|----------|
| 红外服务 | 检测人体靠近/离开 | 软件消抖，灵敏度可调 |
| 播放管理 | 控制音频播放流程 | 指定曲目、音量控制、次数管理 |
| DFPlayer驱动 | UART通信控制 | 指令发送、状态检测（BUSY） |
| 电源管理 | 电池检测与功耗控制 | 低电量保护、休眠唤醒 |
| 配置管理 | 参数存储与读取 | Flash存储，掉电保持 |

### 4.4 低功耗策略

| 状态 | 功耗级别 | 主要措施 |
|------|:--------:|----------|
| 播放 | ~150mA | 全速运行，功放工作 |
| 待机 | ~8mA | MCU降频，关闭非必要外设 |
| 休眠 | ~0.5mA | STOP模式，仅红外中断唤醒 |

## 5. 结构设计概要

### 5.1 外壳方案

- **结构形式**: 上下分体式外壳
- **外形尺寸**: 105 × 75 × 42 mm（长×宽×高）
- **壁厚**: 2.0 mm
- **材料**: PLA Basic（3D打印）

### 5.2 功能区布局

```
上盖（正面）:
┌─────────────────────────┐
│  ┌─────┐   ● ●         │  <- 红外窗口 + LED指示灯
│  │     │               │
│                         │
│   ○  ○  ○  ○  ○        │  <- 扬声器出声孔（Φ3mm阵列）
│   ○  ○  ○  ○  ○        │
│   ○  ○  ○  ○  ○        │
│                         │
│      SmokeVoice         │  <- 产品名称
└─────────────────────────┘

下壳（侧面）:
┌──────────────────────────────────┐
│  [USB-A] [开关] [Type-C]         │  <- 接口布局（右侧）
└──────────────────────────────────┘
```

### 5.3 内部布局

| 模块 | 安装位置 | 固定方式 |

## 7. 方案优势

### 7.1 与原方案对比

| 对比项 | 原方案(VS1053+PAM8403+CH376) | 新方案(DFPlayer Mini) |
|--------|------------------------------|----------------------|
| 模块数量 | 3个独立模块 | 1个集成模块 |
| 通信接口 | SPI×2 + 多GPIO | UART 2-3线 |
| 软件开发 | 复杂（文件系统+驱动） | 简单（串口指令） |
| 代码量 | 约3000行 | 约1500行 |
| 单台成本 | 约252元 | 约112元 |
| 开发周期 | 6-8周 | 3-4周 |

### 7.2 核心优势

1. **开发简单**: UART通信替代复杂SPI和USB Host开发
2. **成本降低**: 硬件成本节省约53%，开发成本显著降低
3. **可靠性高**: 集成模块经过市场验证，故障点减少
4. **维护便捷**: U盘更换音频，无需打开设备
