# 新品卷烟推广语音播报器 (SmokeVoice)

## 项目概述

本项目是一个嵌入式硬件+软件项目，旨在开发一款用于卷烟柜台的新品推广语音播报器。当消费者靠近柜台时，设备通过红外感应自动激活，播报卷烟新品（如芙蓉王总裁等）的品规、特点、价格等信息。

**项目状态**: 设计阶段，已完成硬件和软件详细设计文档（DFPlayer方案）。

**方案变更记录**：
- 2026-03-12: 采纳评审建议，改用DFPlayer Mini一体化方案
- 原方案：VS1053B + PAM8403 + CH376S（三模块方案）
- 新方案：DFPlayer Mini（三合一集成方案）
- 变更原因：降低开发难度、缩短开发周期、减少故障点

**目标交付**: 一个功能完整的样品设备，用于课题任务汇报展示，无需量产。

**截止日期**: 2026年5月20日

## 技术架构

### 硬件组成

| 模块 | 选型 | 功能说明 |
|------|------|----------|
| 核心控制 | STM32F103C8T6 小型开发板 | 红外感应信号处理、播报逻辑控制、供电管理 |
| 感应模块 | HC-SR501 人体红外感应传感器 | 检测消费者靠近/远离，感应距离1-3米可调 |
| **音频模块** | **DFPlayer Mini** | **集成 MP3/WAV解码 + 3W功放 + USB接口** |
| 供电模块 | 5V锂电池(18650) + Type-C充电电路 | 主供电方式，预留干电池仓备用 |
| ~~存储接口~~ | ~~USB接口~~ | ~~已集成到DFPlayer~~ |
| 外壳 | 3D打印塑料（PLA Basic） | 精细美观，预留传感器窗口和出声孔 |

**方案变更说明**：
- 原方案：VS1053B解码 + PAM8403功放 + CH376 USB接口（三模块）
- 新方案：DFPlayer Mini（三合一集成模块）
- **优势**：开发难度大幅降低、成本节省约53元、代码量减少约50%

### 软件架构（DFPlayer方案）

```
┌─────────────────────────────────────────────────────────────┐
│                      应用层 (Application)                     │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐   │
│  │ 感应控制逻辑  │  │ 音频管理模块  │  │ 系统设置模块      │   │
│  └──────────────┘  └──────────────┘  └──────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                      服务层 (Services)                        │
│  ┌──────────────┐  ┌──────────────┐                         │
│  │ 红外感应驱动  │  │ DFPlayer驱动  │                         │
│  └──────────────┘  └──────────────┘                         │
├─────────────────────────────────────────────────────────────┤
│                      硬件抽象层 (HAL)                         │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐   │
│  │ GPIO控制      │  │ UART通信     │  │ 电源管理         │   │
│  └──────────────┘  └──────────────┘  └──────────────────┘   │

```

**架构简化**：
- 无需SPI驱动（原VS1053/CH376使用SPI）
- 无需USB Host驱动（DFPlayer内置）
- 无需文件系统（DFPlayer内置FAT32支持）
- 仅需UART串口驱动，代码量约1500行（原方案约3000行）

## 核心功能需求

### 1. 红外感应触发
- 检测到人体靠近（1-3米范围）时启动音频播放
- 播放完成后进入待机状态
- 检测到人体远离后立即停止播报并进入低功耗休眠

### 2. 音频管理
- **音频播放**: 通过U盘导入预录音频文件（MP3/WAV格式），按顺序播放
- **播放次数**: 默认播放1遍

### 3. 系统设置
- 感应灵敏度调节
- 播报音量调节
- 播放次数设置
- 通过物理按键或串口进行配置

### 4. 低功耗管理
- 休眠唤醒机制
- 待机时关闭非必要模块供电
- 仅保留红外感应模块低功耗运行

## 技术栈选择

### 方案A: STM32平台
- **MCU**: STM32F103 或 STM32F4系列
- **开发环境**: STM32CubeIDE / Keil MDK
- **编程语言**: C/C++
- **优势**: 稳定性高，外设丰富，工业级可靠性

### 方案B: ESP32平台
- **MCU**: ESP32-S3 或 ESP32-C3
- **开发环境**: Arduino IDE / PlatformIO / ESP-IDF
- **编程语言**: C/C++ (Arduino框架) 或 C (ESP-IDF)
- **优势**: 集成WiFi/蓝牙（未来可扩展），社区资源丰富，开发便捷

**当前推荐**: 考虑到开发周期和样品制作，建议采用 **ESP32 + Arduino框架** 方案，便于快速原型验证。

## 关键硬件接线参考（DFPlayer方案）

```
STM32F103            HC-SR501        DFPlayer Mini
─────────────────────────────────────────────────────
5V         ────────> VCC             VCC
GND        ────────> GND             GND
PB0        <────────OUT
PA9 (TX)   ────────────────────────> RX
PA10 (RX)  <────────────────────────TX (可选)
PB12       <────────────────────────BUSY
                           SPK1 ────> 扬声器+
                           SPK2 ────> 扬声器-
                           USB+ ────> U盘 D+
                           USB- ────> U盘 D-
```

**接线简化说明**：
- 仅需3根信号线：TX、RX（可选）、BUSY
- 无需SPI（4线）和USB Host接口
- DFPlayer直接驱动扬声器，无需外接功放

## 开发规范建议

### 代码风格
- 使用清晰的函数命名，采用 `snake_case` 风格
- 关键逻辑添加中文注释，方便后续维护
- 模块化设计，各功能模块解耦

### 示例代码结构 (STM32/DFPlayer)

```cpp
// main.c - DFPlayer方案简化示例
#include "pir_sensor.h"
#include "dfplayer_driver.h"
#include "power_manager.h"

// DFPlayer UART指令示例
// 播放第1首: 7E FF 06 0F 00 00 01 FE E5 EF
// 设置音量: 7E FF 06 06 00 00 14 FE D6 EF (音量20)

void setup() {
    // 初始化调试串口
    usart2_init(115200);
    
    // 初始化红外感应
    pir_sensor_init();
    
    // 初始化DFPlayer (UART1, 9600bps)
    dfplayer_init(9600);
    dfplayer_set_volume(20);  // 设置音量
    
    // 初始化电源管理
    power_manager_init();
}

void loop() {
    // 检测人体靠近
    if (pir_sensor_detected()) {
        // 播放第1首（通过UART发送指令）
        dfplayer_play_track(1);
    }
    
    // 检测播放结束（通过BUSY引脚）
    if (!dfplayer_is_playing()) {
        // 播放结束处理
    }
    
    // 检查电池电量
    power_manager_check_battery();
    
    delay(100);
}
```
