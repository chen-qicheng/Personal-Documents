## 1. 概述

### 1.1 设计目标

本文档详细描述语音播报器嵌入式软件的设计方案，包括系统架构、模块划分、状态机设计、任务调度等，为软件开发提供完整的技术指导。

### 1.2 软件需求回顾

| 需求编号 | 功能需求 | 优先级 |
|:--------:|----------|:------:|
| R1 | 红外感应触发音频播放 | 高 |
| R2 | 支持MP3/WAV音频文件播放 | 高 |
| R3 | 支持U盘导入播放列表 | 高 |
| R4 | 人离开后停止播放并休眠 | 高 |
| R5 | 可调节感应灵敏度 | 中 |
| R6 | 可调节播放音量 | 中 |
| R7 | 可调节播放次数 | 中 |
| R8 | 低功耗待机 | 高 |
| R9 | 电池电量检测 | 中 |

### 1.3 开发环境

- **开发工具**: STM32CubeIDE / Keil MDK-ARM
- **固件库**: STM32标准外设库 (SPL) / HAL库
- **编程语言**: C (C99标准)
- **调试工具**: J-Link / ST-Link
- **版本控制**: Git

## 3. 系统状态机设计

### 3.1 主状态机

```
                    ┌─────────────────────────────────────────┐
                    │                                         │
                    v                                         │
┌──────────┐   上电/复位   ┌──────────┐   检测到人体   ┌──────────┐
│          │──────────────>│          │───────────────>│          │
│  POWER_ON│               │  STANDBY │                │  PLAYING │
│  上电初始化│               │  待机检测 │                │  播放中  │
│          │<──────────────│          │<───────────────│          │
└──────────┘  初始化失败   └────┬─────┘   人离开/播放完 └────┬─────┘
                                │                            │
               低电量/长时间无活动│                            │ 播放完成
               或按键休眠       │                            │ 人未离开
                                v                            │
                         ┌──────────┐                        │
                         │          │<───────────────────────┘
                         │  SLEEP   │    定时唤醒检测
                         │  深度休眠 │
                         │          │──────────────────┐
                         └────┬─────┘   红外中断唤醒    │
                              └────────────────────────┘
```

### 3.2 状态定义与转换条件

| 状态 | 定义 | 进入条件 | 退出条件 | 功耗级别 |
|------|------|----------|----------|:--------:|
| POWER_ON | 上电初始化 | 系统上电/复位 | 初始化完成 | 高 |
| STANDBY | 待机检测 | 初始化完成/从SLEEP唤醒 | 检测到人体/进入SLEEP | 中 |
| PLAYING | 播放中 | STANDBY检测到人体 | 播放完成/人离开 | 高 |
| SLEEP | 深度休眠 | 低电量/长时间无活动/按键休眠 | 红外中断唤醒 | 低 |

### 3.3 状态转换表

| 当前状态 | 事件 | 下一状态 | 动作 |
|----------|------|----------|------|
| POWER_ON | 初始化成功 | STANDBY | 开启红外检测，DFPlayer就绪 |
| POWER_ON | 初始化失败 | POWER_ON | 重试/报错 |
| STANDBY | 检测到人体 | PLAYING | 发送播放指令，DFPlayer开始播放 |
| STANDBY | 低电量 | SLEEP | 保存状态，进入休眠 |
| STANDBY | 休眠按键 | SLEEP | 保存状态，进入休眠 |
| STANDBY | 5分钟无活动 | SLEEP | 自动进入休眠 |
| PLAYING | 播放完成且人离开 | STANDBY | 等待下一次触发 |
| PLAYING | 播放完成人未离开 | PLAYING | 继续播放（循环或下一首） |
| PLAYING | 人离开 | STANDBY | 发送停止指令 |
| SLEEP | 红外中断 | STANDBY | 快速唤醒，恢复检测 |
| SLEEP | 按键唤醒 | STANDBY | 完全唤醒，恢复检测 |

**简化说明**：
- 无需"加载音频"操作（DFPlayer自动从U盘读取）
- 无需"开启/关闭功放"（DFPlayer内部自动管理）
- 播放控制简化为UART指令发送

#### 4.1.1 功能职责

- 系统初始化和各模块初始化调度

#### 4.1.2 程序流程

```c
void system_init(void) {
    // 1. 硬件初始化
    rcc_init();           // 时钟系统
    gpio_init();          // GPIO配置
    usart_init();         // 调试串口
    
    // 2. 驱动初始化
    spi1_init();          // VS1053 SPI
    spi2_init();          // CH376 SPI
    adc_init();           // 电池检测
    
    // 3. 服务初始化
    logger_init();
    power_manager_init();
    tick_timer_init();
    
    // 4. 功能模块初始化
    pir_service_init();
    vs1053_init();
    ch376_init();
    
    // 5. 应用层初始化
    config_load();        // 加载配置
    playlist_init();      // 初始化播放列表
    audio_manager_init();
    
    // 6. 进入待机状态
    system_set_state(STATE_STANDBY);
}

void main_loop(void) {
    while (1) {
        // 状态机处理
        state_machine_run();
        
        // 事件处理
        event_process();
        
        // 定时任务
        scheduled_tasks();
        
        // 功耗管理
        power_manager_run();
    }
}
```

### 4.2 红外感应服务 (pir_service)

#### 4.2.1 功能说明

- 读取HC-SR501输出状态
- 消抖处理（防止误触发）
- 灵敏度软件调节
- 触发事件通知

#### 4.2.2 消抖算法

```c
#define PIR_DEBOUNCE_TIME    500     // 消抖时间 500ms
#define PIR_DETECT_INTERVAL  100     // 检测间隔 100ms

typedef enum {
    PIR_STATE_IDLE = 0,      // 无人状态
    PIR_STATE_DETECTING,     // 检测中（消抖）
    PIR_STATE_DETECTED,      // 确认有人
    PIR_STATE_LOSING,        // 丢失中（消抖）
} pir_state_t;

typedef struct {
    pir_state_t state;
    uint32_t last_change_time;
    uint8_t stable_count;
    uint8_t is_triggered;
} pir_context_t;

void pir_service_run(void) {
    uint8_t raw_status = gpio_read(PIR_PIN);
    uint32_t now = tick_get();
    
    switch (pir_ctx.state) {
        case PIR_STATE_IDLE:
            if (raw_status == 1) {
                pir_ctx.state = PIR_STATE_DETECTING;
                pir_ctx.last_change_time = now;
            }
            break;
            
        case PIR_STATE_DETECTING:
            if (raw_status == 0) {
                pir_ctx.state = PIR_STATE_IDLE;
            } else if (now - pir_ctx.last_change_time >= PIR_DEBOUNCE_TIME) {
                pir_ctx.state = PIR_STATE_DETECTED;
                pir_ctx.is_triggered = 1;
                event_post(EVENT_PIR_TRIGGERED);
            }
            break;
            
        case PIR_STATE_DETECTED:
            if (raw_status == 0) {
                pir_ctx.state = PIR_STATE_LOSING;
                pir_ctx.last_change_time = now;
            }
            break;
            
        case PIR_STATE_LOSING:
            if (raw_status == 1) {
                pir_ctx.state = PIR_STATE_DETECTED;
            } else if (now - pir_ctx.last_change_time >= PIR_DEBOUNCE_TIME) {
                pir_ctx.state = PIR_STATE_IDLE;
                pir_ctx.is_triggered = 0;
                event_post(EVENT_PIR_LOST);
            }
            break;
    }
}
```

### 4.4 DFPlayer驱动 (dfplayer_driver)

#### 4.4.1 功能说明

- UART串口通信控制
- 播放控制指令发送
- 播放状态查询
- 音量设置

**与VS1053对比**：
- VS1053：需要SPI通信，持续发送音频数据流，复杂的寄存器配置
- DFPlayer：简单的UART指令，模块内部处理所有解码和播放

#### 4.4.2 指令定义

```c
// DFPlayer指令码
#define DFPLAYER_CMD_NEXT          0x01   // 下一首
#define DFPLAYER_CMD_PREV          0x02   // 上一首
#define DFPLAYER_CMD_TRACK         0x03   // 指定曲目(0-65535)
#define DFPLAYER_CMD_VOLUME_UP     0x04   // 音量加
#define DFPLAYER_CMD_VOLUME_DOWN   0x05   // 音量减
#define DFPLAYER_CMD_VOLUME_SET    0x06   // 设置音量(0-30)
#define DFPLAYER_CMD_EQ            0x07   // 设置EQ
#define DFPLAYER_CMD_PLAY_MODE     0x08   // 播放模式
#define DFPLAYER_CMD_PLAY_SRC      0x09   // 播放源
#define DFPLAYER_CMD_STANDBY       0x0A   // 待机
#define DFPLAYER_CMD_NORMAL        0x0B   // 恢复正常
#define DFPLAYER_CMD_RESET         0x0C   // 复位
#define DFPLAYER_CMD_PLAY          0x0D   // 播放
#define DFPLAYER_CMD_PAUSE         0x0E   // 暂停
#define DFPLAYER_CMD_TRACK_FOLDER  0x0F   // 指定文件夹曲目
#define DFPLAYER_CMD_STATUS        0x42   // 查询状态
#define DFPLAYER_CMD_VOLUME        0x43   // 查询音量
#define DFPLAYER_CMD_TRACK_COUNT   0x48   // 查询曲目总数

// 反馈标志
#define DFPLAYER_FEEDBACK_OK       0x41   // 执行成功
#define DFPLAYER_FEEDBACK_ERR      0x40   // 执行错误
```

#### 4.4.3 核心函数

```c
// 初始化
void dfplayer_init(void) {
    // 初始化UART1 (PA9/PA10), 9600bps
    uart1_init(9600);
    
    // 初始化BUSY引脚 (PB12, 上拉输入)
    gpio_init_input(BUSY_PIN, GPIO_PULLUP);
    
    // 等待模块就绪
    delay_ms(1000);
    
    // 设置默认音量
    dfplayer_set_volume(20);  // 中等音量
    
    // 设置播放源为U盘
    dfplayer_send_cmd(DFPLAYER_CMD_PLAY_SRC, 0, 1);  // 1=U盘
}

// 发送指令
void dfplayer_send_cmd(uint8_t cmd, uint8_t param_h, uint8_t param_l) {
    uint8_t packet[10];
    
    packet[0] = 0x7E;       // 起始码
    packet[1] = 0xFF;       // 版本
    packet[2] = 0x06;       // 长度
    packet[3] = cmd;        // 命令
    packet[4] = 0x00;       // 反馈标志 (0=不需要反馈)
    packet[5] = param_h;    // 参数高字节
    packet[6] = param_l;    // 参数低字节
    
    // 计算校验和
    uint16_t checksum = 0;
    for (int i = 1; i <= 6; i++) {
        checksum += packet[i];
    }
    checksum = 0xFFFF - checksum + 1;
    
    packet[7] = (checksum >> 8) & 0xFF;   // 校验和高字节
    packet[8] = checksum & 0xFF;          // 校验和低字节
    packet[9] = 0xEF;                     // 结束码
    
    // 发送数据
    uart1_send_array(packet, 10);
}

// 播放指定曲目 (1-65535)
void dfplayer_play_track(uint16_t track_num) {
    dfplayer_send_cmd(DFPLAYER_CMD_TRACK, 
                     (track_num >> 8) & 0xFF, 
                     track_num & 0xFF);
}

// 设置音量 (0-30)
void dfplayer_set_volume(uint8_t volume) {
    if (volume > 30) volume = 30;
    dfplayer_send_cmd(DFPLAYER_CMD_VOLUME_SET, 0, volume);
}

// 检测播放状态 (通过BUSY引脚)
uint8_t dfplayer_is_playing(void) {
    // BUSY引脚低电平 = 正在播放
    return (gpio_read(BUSY_PIN) == 0);
}

// 播放控制
void dfplayer_play(void) {
    dfplayer_send_cmd(DFPLAYER_CMD_PLAY, 0, 0);
}

void dfplayer_pause(void) {
    dfplayer_send_cmd(DFPLAYER_CMD_PAUSE, 0, 0);
}

void dfplayer_next(void) {
    dfplayer_send_cmd(DFPLAYER_CMD_NEXT, 0, 0);
}
```

```c
    static uint8_t was_playing = 0;
    uint8_t is_playing = dfplayer_is_playing();
    
    if (was_playing && !is_playing) {
        // 播放结束，触发事件
        event_post(EVENT_AUDIO_FINISHED);
    }
    was_playing = is_playing;
}
```

```c
// 电池电压ADC检测
#define ADC_BATTERY_CHANNEL    ADC_Channel_0
#define ADC_REF_VOLTAGE        3.3f
#define ADC_MAX_VALUE          4095
#define VOLTAGE_DIVIDER_RATIO  2.0f  // 分压比 1/2

// 电池电量等级
#define BATTERY_LEVEL_HIGH     80    // >80%
#define BATTERY_LEVEL_MEDIUM   40    // 40-80%
#define BATTERY_LEVEL_LOW      20    // 20-40%
#define BATTERY_LEVEL_CRITICAL 10    // <20%

typedef enum {
    BATTERY_FULL = 0,
    BATTERY_GOOD,
    BATTERY_LOW,
    BATTERY_CRITICAL,
    BATTERY_EMPTY
} battery_level_t;

// 获取电池电压 (V)
float battery_get_voltage(void) {
    uint16_t adc_value = adc_read(ADC_BATTERY_CHANNEL);
    float voltage = (adc_value * ADC_REF_VOLTAGE / ADC_MAX_VALUE) 
                   * VOLTAGE_DIVIDER_RATIO;
    return voltage;
}

// 获取电池电量百分比 (0-100)
uint8_t battery_get_percentage(void) {
    float voltage = battery_get_voltage();
    
    // 线性映射: 3.0V(0%) ~ 4.2V(100%)
    if (voltage >= 4.2f) return 100;
    if (voltage <= 3.0f) return 0;
    
    uint8_t percentage = (uint8_t)((voltage - 3.0f) / 1.2f * 100);
    return percentage;
}

// 获取电量等级
battery_level_t battery_get_level(void) {
    uint8_t pct = battery_get_percentage();
    
    if (pct >= BATTERY_LEVEL_HIGH) return BATTERY_FULL;
    if (pct >= BATTERY_LEVEL_MEDIUM) return BATTERY_GOOD;
    if (pct >= BATTERY_LEVEL_LOW) return BATTERY_LOW;
    if (pct >= BATTERY_LEVEL_CRITICAL) return BATTERY_CRITICAL;
    return BATTERY_EMPTY;
}
```

## 5. 任务调度设计

### 5.1 时间片轮询架构

采用非抢占式轮询调度，简化设计，避免RTOS开销。

```c
// 任务定义
typedef struct {
    void (*task_func)(void);
    uint32_t interval_ms;
    uint32_t last_run;
    uint8_t enabled;
} task_t;

// 任务列表
task_t g_tasks[] = {
    {pir_service_run,       50,   0, 1},  // 红外检测 50ms
    {audio_manager_run,     10,   0, 1},  // 音频流 10ms
    {power_manager_run,     1000, 0, 1},  // 电源管理 1s
    {battery_monitor_run,   5000, 0, 1},  // 电量检测 5s
    {led_indicator_run,     100,  0, 1},  // LED指示 100ms
};

#define TASK_COUNT    (sizeof(g_tasks) / sizeof(task_t))

// 调度器主循环
void scheduler_run(void) {
    uint32_t now = tick_get();
    
    for (uint8_t i = 0; i < TASK_COUNT; i++) {
        if (g_tasks[i].enabled && 
            (now - g_tasks[i].last_run) >= g_tasks[i].interval_ms) {
            g_tasks[i].task_func();
            g_tasks[i].last_run = now;
        }
    }
}
```

### 5.2 任务优先级与时序

| 任务 | 周期 | 执行时间 | 优先级 |
|------|:----:|:--------:|:------:|
| 音频流喂数据 | 10ms | <1ms | 最高 |
| 红外检测 | 50ms | <0.5ms | 高 |
| LED指示 | 100ms | <0.1ms | 低 |
| 电源管理 | 1000ms | <1ms | 中 |
| 电量检测 | 5000ms | <0.5ms | 低 |
| 文件系统 | 按需 | 变化 | 中 |

### 5.3 中断处理

| 中断源 | 优先级 | 处理内容 |
|--------|:------:|----------|
| SysTick | 最高 | 系统时基 |
| SPI1 | 高 | VS1053数据传输 |
| SPI2 | 高 | CH376通信 |
| EXTI0 (PIR) | 中 | 红外唤醒 |
| USART1 | 低 | 调试输出 |
| ADC | 低 | 电池检测完成 |

### 8.2 存储方案

### 8.3 配置接口

```c
// 加载配置
void config_load(void) {
    // 从Flash读取
    memcpy(&g_config, (void*)CONFIG_FLASH_ADDR, sizeof(system_config_t));
    
    // 校验
    if (g_config.magic != CONFIG_MAGIC || 
        g_config.version != CONFIG_VERSION) {
        config_load_default();  // 使用默认配置
    }
}

// 保存配置
void config_save(void) {
    FLASH_Unlock();
    FLASH_ErasePage(CONFIG_FLASH_ADDR);
    
    uint32_t *src = (uint32_t*)&g_config;
    for (uint8_t i = 0; i < sizeof(g_config) / 4; i++) {
        FLASH_ProgramWord(CONFIG_FLASH_ADDR + i*4, src[i]);
    }
    
    FLASH_Lock();
}

// 恢复默认
void config_load_default(void) {
    g_config.version = CONFIG_VERSION;
    g_config.volume = 70;
    g_config.sensitivity = 50;
    g_config.play_mode = PLAYMODE_SEQUENCE;
    g_config.max_play_count = 1;
    g_config.standby_timeout = 300;  // 5分钟
    g_config.magic = CONFIG_MAGIC;
}
```
