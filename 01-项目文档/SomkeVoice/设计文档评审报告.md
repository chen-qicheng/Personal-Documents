#### 问题 #4: 电池保护电路设计存在风险

| 项目 | 内容 |
|------|------|
| **问题描述** | 6.3.3节保护电路原理图中，DW01A的VM引脚(第5脚)直接连接B-（电池负极），但缺少采样电阻。标准应用中VM引脚应通过一个小电阻（如1kΩ）连接，用于检测过流。 |
| **影响程度** | 高 - 可能导致过流保护功能失效或误触发 |
| **修改意见** | 在VM引脚与B-之间增加1kΩ采样电阻，参考DW01A数据手册典型应用电路。同时增加100nF滤波电容到VM引脚，提高抗干扰能力。 |

#### 问题 #5: 电压检测电路精度不足

| 项目 | 内容 |
|------|------|
| **问题描述** | 6.6.1节电压检测电路使用100kΩ+100kΩ分压电阻，虽然功耗低，但STM32 ADC输入阻抗要求源阻抗≤50kΩ（根据数据手册），高阻抗会导致采样误差。 |
| **影响程度** | 中 - 电池电量显示可能不准确 |
| **修改意见** | 方案一：分压电阻改为47kΩ+47kΩ，降低源阻抗<br>方案二：保留100kΩ分压，但在ADC输入端增加运放跟随器（如LMV321）<br>方案三：增加ADC采样时间设置，使用更长的采样周期补偿高阻抗 |

#### 问题 #6: 功放使能控制逻辑不明确

| 项目 | 内容 |
|------|------|
| **问题描述** | 3.2节引脚分配表中PB12用于"功放使能"，但文档中未明确使能逻辑是高电平有效还是低电平有效。PAM8403模块通常有SHUTDOWN引脚（低电平关断）。 |
| **影响程度** | 低 - 可能导致软件控制逻辑错误 |
| **修改意见** | 明确功放使能逻辑：<br>- 若使用PAM8403的SHUTDOWN引脚：低电平=关断，高电平=工作<br>- 若使用MOS管控制电源：高电平=导通<br>在硬件设计文档中增加功放电源控制电路图，标注逻辑电平。 |

#### 问题 #7: 缺少ESD保护设计

| 项目 | 内容 |
|------|------|
| **问题描述** | USB接口、按键等用户可接触部位缺少ESD（静电放电）保护设计。柜台环境可能存在静电积累风险。 |
| **影响程度** | 中 - 可能导致设备静电损坏 |
| **修改意见** | 在以下位置增加ESD保护器件：<br>1. USB D+/D-线：增加TVS二极管阵列（如USBLC6-2SC6）<br>2. 按键输入：增加100nF电容滤波+TVS管<br>3. 红外传感器输出：增加100Ω串联电阻+TVS管 |

#### 问题 #8: 晶振电路参数缺失

| 项目 | 内容 |
|------|------|
| **问题描述** | 3.3节最小系统电路提到使用8MHz主晶振和32.768kHz RTC晶振，但未给出晶振选型参数（负载电容、ESR、精度）。 |
| **影响程度** | 低 - 可能导致晶振不起振或频率偏差 |
| **修改意见** | 补充晶振选型参数：<br>- 主晶振：8MHz，20pF负载电容，±20ppm精度，HC-49S封装<br>- RTC晶振：32.768kHz，12.5pF负载电容，±20ppm精度，3×8mm圆柱 |

#### 问题 #9: 扬声器线缆规格未指定

| 项目 | 内容 |
|------|------|
| **问题描述** | 5.3.2节扬声器连接未指定线缆规格，PAM8403输出为D类功放，线缆选择不当可能影响音质或产生EMI。 |

### 2.3 软件详细设计说明书问题

#### 问题 #10: 状态机逻辑矛盾

| 项目 | 内容 |
|------|------|
| **问题描述** | 3.1节状态机图中，PLAYING状态"播放完成人未离开"会继续保持PLAYING状态，但3.3节状态转换表显示"播放完成人未离开"会"继续播放下一首"，两者描述不一致。另外，需求中要求"播放1遍左右"，但状态机设计支持循环播放，存在需求与设计不符。 |
| **影响程度** | 高 - 核心功能逻辑混乱 |
| **修改意见** | 统一状态机逻辑：<br>1. 明确播放模式：单次播放模式 vs 循环播放模式<br>2. 修改状态转换表，与状态图保持一致<br>3. 增加播放计数器逻辑，达到设定次数后即使人未离开也返回STANDBY<br>4. 在PLAYING状态增加"播放次数已达上限"的退出条件 |

#### 问题 #11: 低功耗唤醒机制设计缺陷

| 项目 | 内容 |
|------|------|
| **问题描述** | 4.6.3节休眠唤醒设计中，PIR信号连接到EXTI线，但HC-SR501的输出是电平信号（高电平持续），不是脉冲信号。STM32从STOP模式唤醒需要边沿触发，电平信号可能导致无法正确唤醒或频繁唤醒。 |
| **影响程度** | 高 - 可能导致唤醒功能失效 |
| **修改意见** | 修改唤醒电路设计：<br>方案一：在PIR输出和MCU之间增加单稳态触发器（如使用555定时器或门电路），将电平信号转换为脉冲信号<br>方案二：使用WKUP引脚（PA0）作为唤醒源，支持电平唤醒<br>方案三：改为定期唤醒模式（RTC定时唤醒），唤醒后检测PIR状态 |

#### 问题 #12: 音频缓冲区大小可能不足 **[已通过方案变更解决]**

| 项目 | 内容 |
|------|------|
| **问题描述** | 4.3.3节定义AUDIO_BUFFER_SIZE为4096字节（4KB），但VS1053在较高码率下（如320kbps MP3）需要更大的缓冲。计算：320kbps = 40KB/s，若SPI传输有延迟，4KB缓冲只能维持约100ms。 |
| **影响程度** | 中 - 高码率音频可能出现卡顿 |
| **解决方案** | ✅ **已通过方案变更解决**：改用DFPlayer Mini方案后，音频缓冲完全由模块内部处理，MCU无需管理音频缓冲区，从根本上解决了该问题。 |
| **修改意见** | 新方案DFPlayer内置音频缓冲，MCU只需发送控制指令，无需处理音频数据流。 |

#### 问题 #13: 文件系统错误处理不完善 **[已通过方案变更解决]**

| 项目 | 内容 |
|------|------|
| **问题描述** | 4.5.3节文件扫描代码中，当U盘文件数量超过MAX_PLAYLIST_SIZE（50）时直接break，但没有错误提示或处理逻辑。用户可能不知道部分文件未被加载。 |
| **影响程度** | 低 - 用户体验问题 |
| **解决方案** | ✅ **已通过方案变更解决**：改用DFPlayer Mini方案后，文件系统完全由模块内部管理，MCU无需扫描文件列表，支持最大65535首曲目，不存在数量限制问题。 |
| **修改意见** | 新方案DFPlayer自动按物理顺序管理文件，通过曲目号(1-65535)直接访问，无需文件扫描。 |

#### 问题 #14: 缺少看门狗设计

| 项目 | 内容 |
|------|------|
| **问题描述** | 软件架构中提到了IWDG驱动，但在主控制逻辑和状态机设计中未明确看门狗的喂狗策略。长时间播放或文件读取时若未喂狗可能导致误复位。 |
| **影响程度** | 中 - 可能导致系统不稳定 |
| **修改意见** | 明确看门狗策略：<br>1. 设置看门狗超时时间为1-2秒<br>2. 在主循环中定期喂狗<br>3. 在长时间操作（如文件读取）期间使用窗口看门狗或临时禁用策略<br>4. 在看门狗复位后保存错误日志，便于问题定位 |

#### 问题 #15: 配置参数存储无校验机制

| 项目 | 内容 |
|------|------|
| **问题描述** | 8.3节配置存储仅使用magic和version校验，缺少数据完整性校验（如CRC）。Flash写入过程中断电可能导致配置损坏但校验通过。 |
| **影响程度** | 中 - 配置可能损坏导致异常行为 |

### 2.4 结构设计说明书问题

#### 问题 #16: 红外模块倾斜角度设计验证不足

| 项目 | 内容 |
|------|------|
| **问题描述** | 4.2.6节红外模块采用15°倾斜设计，但未说明该角度的设计依据。HC-SR501的感应角度为120°锥角，15°倾斜可能导致感应盲区或覆盖范围不符合柜台场景需求。 |
| **影响程度** | 中 - 可能影响感应效果 |
| **修改意见** | 补充设计验证：<br>1. 说明15°角度的计算依据（柜台高度、探测距离）<br>2. 建议制作角度可调支架原型，实际测试后确定最佳角度<br>3. 或设计为角度可调结构（如多档位卡槽），便于现场调试 |

#### 问题 #17: 扬声器出声孔开孔率计算有误

| 项目 | 内容 |
|------|------|
| **问题描述** | 4.1.4节扬声器出声孔开孔率计算为28%，但按照给出的参数（Φ3mm孔，5mm间距，90个孔）重新计算：<br>- 单孔面积 = π×1.5² = 7.07mm²<br>- 总孔面积 = 90×7.07 = 636mm²<br>- 区域面积 = 60×45 = 2700mm²<br>- 开孔率 = 636/2700 = 23.5%<br>与文档中的28%不符。 |
| **影响程度** | 低 - 文档数据错误 |
| **修改意见** | 修正开孔率为23.5%，或调整孔间距至4.3mm以达到28%开孔率。同时确认23.5%开孔率是否满足声学要求（一般要求>20%）。 |

#### 问题 #18: 电池仓尺寸与电池规格不匹配

| 项目 | 内容 |
|------|------|
| **问题描述** | 4.2.4节电池仓内尺寸为70×22×20mm，但18650电池直径为18mm，长度65mm。若电池仓高度仅20mm，而电池直径18mm，加上电池盒/保护板厚度，空间余量不足。 |
| **影响程度** | 中 - 电池可能无法放入或取出困难 |
| **修改意见** | 重新核算电池仓尺寸：<br>- 18650电池直径：18mm（实际带外皮约18.5mm）<br>- 建议仓内宽度：≥22mm（含余量）<br>- 建议仓内长度：≥70mm<br>- 建议仓内高度：≥25mm（考虑电池盒/保护板） |

#### 问题 #19: 缺少防水防尘设计

| 项目 | 内容 |
|------|------|
| **问题描述** | 柜台环境可能有灰尘或液体溅落风险，但结构设计未考虑防水防尘（IP等级）。 |
| **影响程度** | 低 - 可能影响设备寿命 |
| **修改意见** | 增加基础防护设计：<br>1. USB接口、充电口增加橡胶塞<br>2. 上下盖接缝处设计防水槽<br>3. 扬声器出声孔内侧增加防水透气膜（如ePTFE膜）<br>4. 红外窗口采用密封设计<br>目标：达到IP30或IP41防护等级 |

### 2.5 测试用例文档问题

#### 问题 #20: 电池安全测试缺少详细操作规程

| 项目 | 内容 |
|------|------|
| **问题描述** | 7.3节电池安全测试（TC-SAF-001~006）虽标注了安全警告，但缺少详细的测试操作规程、应急处理步骤、测试设备要求。特别是TC-SAF-004短路保护测试为高危测试，风险极大。 |

#### 问题 #21: 缺少音频质量客观测试项

|------|------|

## 3. 整体优化建议

### 3.1 硬件设计优化

1. **方案简化建议**
   - 考虑使用集成度更高的方案，如DFPlayer Mini MP3模块替代VS1053+PAM8403，降低开发难度
   - 或使用带USB Host功能的STM32F4系列，省去CH376模块

2. **电源优化**
   - 增加电源路径管理（Power Path），实现边充边用功能
   - 考虑使用专用电源管理IC（如IP5306）替代分立方案，简化设计

3. **EMC优化**
   - 增加整体屏蔽设计，特别是音频部分
   - USB接口增加共模电感，抑制共模噪声

### 3.2 软件设计优化

1. **固件升级功能**
   - 增加USB固件升级功能（DFU），便于后续维护
   - 或增加串口ISP预留接口

2. **音频解码优化**
   - 考虑支持更多音频格式（如AAC、OGG）
   - 增加音频淡入淡出效果，提升用户体验

3. **日志系统**
   - 增加运行日志记录功能，便于问题追溯
   - 可考虑使用外部Flash或EEPROM存储日志

### 3.3 结构设计优化

1. **模块化设计**
   - 主板采用子板+底板结构，便于维修更换
   - 电池设计为可快速更换结构

2. **散热优化**
   - 功放模块增加散热片或散热孔
   - 考虑外壳材质导热性

3. **安装便利性**
   - 增加柜台安装支架设计（如L型支架、双面胶槽）
   - 设计防盗锁孔

### 3.4 测试优化

1. **自动化测试**
   - 开发自动化测试程序，提高测试效率
   - 使用Python+串口实现自动化测试脚本

2. **测试工装**
   - 制作专用测试工装（假电池、模拟负载）
   - 设计测试夹具，提高测试一致性
