## 1. 概述

### 1.1 设计目标

本文档详细描述新品卷烟推广语音播报器的硬件设计方案，涵盖各功能模块的选型依据、电路设计、接口定义及装配要求，为硬件采购、组装调试提供技术指导。

### 1.2 设计原则

- **可靠性**: 选用成熟稳定的工业级元器件
- **低功耗**: 支持休眠唤醒机制，延长电池续航
- **模块化**: 各功能模块独立设计，便于调试和维护
- **成本控制**: 在满足功能需求前提下优化成本

### 1.3 参考标准

- GB/T 17626 电磁兼容试验和测量技术
- GB 4943.1 信息技术设备安全标准
- IPC-2221 印刷电路板设计通用标准

## 3. 核心控制模块设计

### 3.1 MCU选型

**选用型号**: STM32F103C8T6 (STM32最小系统板)

**选型理由**:
- 72MHz主频，性能满足音频控制需求
- 64KB Flash + 20KB RAM，存储空间充足
- 丰富外设接口：SPI、I2S、USART、USB等
- 低功耗模式支持，适合电池供电场景
- 开发资料丰富，社区支持好
- 成本低（约15-25元/片）

## 4. 红外感应模块设计

### 4.1 模块选型

**选用型号**: HC-SR501 人体红外感应模块

**技术参数**:
| 参数 | 规格 |
|------|------|
| 工作电压 | DC 4.5V ~ 20V（建议5V） |
| 静态电流 | <50uA |
| 电平输出 | 高电平3.3V / 低电平0V |
| 延时时间 | 0.5s ~ 200s（可调） |
| 封锁时间 | 2.5s（默认，可定制） |
| 感应范围 | <120°锥角，距离3米内 |
| 工作温度 | -15℃ ~ +70℃ |
| 触发方式 | L不可重复触发 / H可重复触发 |

### 4.2 电路连接

```
HC-SR501 模块
┌─────────────────┐
│  VCC  GND  OUT  │
│   │    │    │   │
└───┴────┴────┴───┘
    │    │    │
    │    │    └────────────────────┐
    │    │                         │
    │    └───────────┐             │
    │                │             │
    │               GND            │
    │                              │
    └────────────────┐             │
                     │             │
    5V供电 ──────────┴─────────────┤
                                   │
                          ┌────────┴────┐
                          │   STM32     │
                          │   PB0       │
                          │ (GPIO输入)  │
                          └─────────────┘
```

### 4.3 灵敏度调节

模块板载两个电位器:

1. **距离调节电位器 (SENS)**: 顺时针旋转增加感应距离（最大约7米）
   - 建议设置：感应距离 1.5-2 米（柜台场景）

2. **延时调节电位器 (TIME)**: 顺时针旋转增加输出延时
   - 建议设置：延时 2-5 秒，避免频繁触发

### 4.4 触发方式设置

模块背部跳线帽设置:
- **H (可重复触发)**: 感应到人体后持续输出高电平，直到人离开
  - **推荐采用此模式**
- **L (不可重复触发)**: 感应后输出固定延时的高电平，期间不再检测

### 4.5 与MCU接口设计

```c
// 软件配置建议
#define PIR_SENSOR_PIN    GPIO_Pin_0    // PB0
#define PIR_SENSOR_PORT   GPIOB

// 输入配置（上拉输入）
GPIO_InitStructure.GPIO_Pin = PIR_SENSOR_PIN;
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;  // 上拉输入
GPIO_Init(PIR_SENSOR_PORT, &GPIO_InitStructure);

// 读取状态
uint8_t pir_detected = GPIO_ReadInputDataBit(PIR_SENSOR_PORT, PIR_SENSOR_PIN);
```

## 5. 音频播放模块设计

### 5.1 方案选择

采用 **DFPlayer Mini MP3模块** 一体化方案

**方案优势**:
- 集成度高：MP3解码 + 3W功放 + U盘/TF卡接口三合一
- 接口简单：仅需UART串口通信（2线），无需复杂SPI
- 开发便捷：串口指令控制，无需处理底层解码和文件系统
- 成本低廉：模块价格约8-12元，替代原VS1053B+PAM8403+CH376S（约63元）
- 稳定可靠：市场验证成熟，社区资源丰富

**功能特性**:
- 支持MP3、WAV格式音频文件
- 支持U盘或TF卡存储（最大32GB，FAT32格式）
- 内置3W功放，可直接驱动4Ω扬声器
- UART串口控制（9600bps），指令简单
- 支持指定曲目播放、音量调节（0-30级）、播放/暂停/停止等

### 5.2 DFPlayer Mini模块接口

#### 5.2.1 引脚定义

| 引脚 | 名称 | 类型 | 说明 |
|:----:|------|:----:|------|
| 1 | VCC | 电源 | 供电输入（4.5-5.5V，建议5V） |
| 2 | GND | 电源 | 电源地 |
| 3 | TX | 输出 | UART发送（接MCU的RX） |
| 4 | RX | 输入 | UART接收（接MCU的TX） |
| 5 | BUSY | 输出 | 播放状态（低电平=播放中，高电平=空闲） |
| 6 | USB+ | USB | USB数据正（接U盘D+） |
| 7 | USB- | USB | USB数据负（接U盘D-） |
| 8 | DAC_R | 模拟输出 | 右声道DAC输出（可选，未用） |
| 9 | DAC_L | 模拟输出 | 左声道DAC输出（可选，未用） |
| 10 | SPK1 | 输出 | 扬声器正极（3W功放输出+） |
| 11 | SPK2 | 输出 | 扬声器负极（3W功放输出-） |

#### 5.2.2 与STM32连接

```
STM32F103                    DFPlayer Mini
┌─────────┐                 ┌─────────────┐
│  PA9    │────────────────>│ 4 RX        │  <-- UART_TX (MCU→DFPlayer)
│  PA10   │<────────────────│ 3 TX        │  <-- UART_RX (可选，接收状态)
│  PB12   │<────────────────│ 5 BUSY      │  <-- 播放状态检测
│         │                 │             │
│  5V     │────────────────>│ 1 VCC       │  <-- 模块供电
│  GND    │────────────────>│ 2 GND       │
└─────────┘                 │             │
                            │ 10 SPK1     │──────> 扬声器+
                            │ 11 SPK2     │──────> 扬声器-
                            │             │
                            │ 6 USB+      │<─────── U盘D+
                            │ 7 USB-      │<─────── U盘D-
                            └─────────────┘
```

**接线说明**:
- **UART通信**：PA9(TX)接DFPlayer的RX，PA10(RX)接DFPlayer的TX
- **BUSY检测**：PB12配置为上拉输入，检测播放状态（低电平=正在播放）
- **电源**：5V供电，与系统主电源共用
- **扬声器**：直接接SPK1和SPK2，无需外接功放
- **U盘**：USB+和USB-直接接U盘接口

### 5.3 UART通信协议

#### 5.3.1 通信参数

| 参数 | 设置 |
|------|------|
| 波特率 | 9600 bps |
| 数据位 | 8 bit |
| 停止位 | 1 bit |
| 校验位 | 无（None） |
| 流控制 | 无 |

#### 5.3.2 指令格式

```
[起始码] [版本] [长度] [命令] [反馈] [参数高] [参数低] [校验和] [结束码]
  0x7E    0xFF   0x06   CMD    0x00   PARAM_H  PARAM_L   CHECKSUM   0xEF
```

**校验和计算**：
```
CHECKSUM = 0xFF - (版本 + 长度 + 命令 + 反馈 + 参数高 + 参数低) + 1
```

#### 5.3.3 常用指令集

| 命令码 | 功能 | 参数说明 |
|:------:|------|----------|
| 0x0D | 播放 | 无参数（0x00 0x00） |
| 0x0E | 暂停 | 无参数 |
| 0x0F | 指定曲目 | 参数 = 曲目序号（1-65535） |
| 0x01 | 下一首 | 无参数 |
| 0x02 | 上一首 | 无参数 |
| 0x06 | 音量设置 | 参数 = 音量值（0-30） |
| 0x07 | 音量加 | 无参数 |
| 0x08 | 音量减 | 无参数 |
| 0x0C | 复位 | 无参数 |
| 0x11 | 指定曲目（根目录） | 参数 = 曲目序号 |
| 0x42 | 查询当前状态 | 返回播放状态 |

**示例指令**：
- 播放第1首：`7E FF 06 0F 00 00 01 FE E5 EF`
- 设置音量20：`7E FF 06 06 00 00 14 FE D6 EF`
- 暂停播放：`7E FF 06 0E 00 00 00 FE ED EF`

### 5.4 扬声器选型

**选用规格**: 4Ω 3W 内磁式扬声器

**选型考虑**:
- 阻抗匹配DFPlayer内置功放输出（4Ω推荐）
- 3W功率满足柜台环境（1-2米距离清晰可听）
- 内磁式避免干扰其他电子元件
- 直径建议 40-57mm，便于外壳设计
- 可直接焊接或使用插接件连接

**连接方式**：
```
DFPlayer Mini SPK1 ────────> 扬声器+
DFPlayer Mini SPK2 ────────> 扬声器-
```

### 5.5 存储介质说明

DFPlayer Mini支持两种存储方式：

#### 5.5.1 U盘模式
- 通过USB接口连接U盘
- 支持FAT32格式，最大32GB
- 音频文件放在根目录或文件夹内
- 文件命名建议：`001.mp3`, `002.mp3`...（便于指定播放）

#### 5.5.2 TF卡模式（可选）
- 模块支持TF卡插槽（部分版本）
- 与U盘模式二选一使用
- 使用方式与U盘相同

**推荐使用U盘模式**：方便用户更换音频文件，无需打开设备外壳。

## 5.6 电源与功耗管理

### 5.6.1 电源供电

DFPlayer Mini使用5V供电，与系统主电源共享。

**供电参数**：
| 参数 | 数值 | 说明 |
|------|------|------|
| 工作电压 | 4.5V ~ 5.5V | 推荐5V |
| 工作电流 | <50mA（待机） | 空闲状态 |
| 工作电流 | 100-300mA（播放） | 取决于音量和音频内容 |
| 峰值电流 | <500mA | 大音量输出时 |

**电源连接**：
```
5V系统电源 ──────┬────────> DFPlayer VCC
                │
               === 100uF 滤波电容
                │
               GND ──────> DFPlayer GND
```

### 5.6.2 低功耗控制

DFPlayer Mini上电后自动进入待机状态，可通过以下方式控制功耗：

**方式一：使用BUSY信号检测**
- 播放完成后自动返回低功耗待机
- MCU通过BUSY引脚检测播放状态

**方式二：电源开关控制（可选）**
- 使用MOS管控制DFPlayer电源（PB7控制）
- 长时间不使用时完全断电

```
PB7 ──────┬─────[1kΩ]────┬─────> MOS管栅极
          │              │

### 5.7.1 设计简化说明

### 6.3 电池保护电路（DW01A+FS8205A）

**设计目的**: 为18650锂电池提供过充、过放、过流、短路保护，确保使用安全。

#### 6.3.1 保护电路方案

采用 **DW01A锂电保护IC + FS8205A双MOS** 组合方案：

- **DW01A**: 锂电保护专用芯片，检测电池电压和电流
- **FS8205A**: 集成双N沟道MOS，用于充放电开关控制

#### 6.3.2 保护参数

| 保护类型 | 阈值 | 动作 |
|----------|------|------|
| 过充电保护 | 4.25V ± 0.05V | 切断充电回路 |
| 过充电解除 | 4.05V ± 0.05V | 恢复充电 |
| 过放电保护 | 2.40V ± 0.08V | 切断放电回路 |
| 过放电解除 | 3.00V ± 0.08V | 恢复放电 |
| 过电流保护 | 3A | 切断放电回路 |
| 短路保护 | 负载短路 | 立即切断 |

```

**工作原理**:

- **滤波效果**: 有效滤除电池纹波和开关噪声
- **采样建议**: ADC采用多次采样取平均，提高精度

#### 6.6.2 ADC采样软件建议

```c
// 电池电压检测 (带软件滤波)
#define ADC_BATTERY_CHANNEL    ADC_Channel_0
#define ADC_SAMPLE_COUNT       16      // 采样次数
#define ADC_DISCARD_COUNT      4       // 舍弃最大最小值各2个

float battery_get_voltage_filtered(void) {
    uint16_t samples[ADC_SAMPLE_COUNT];
    uint32_t sum = 0;
    uint16_t max_val = 0, min_val = 4095;
    
    // 多次采样
    for (int i = 0; i < ADC_SAMPLE_COUNT; i++) {
        samples[i] = adc_read(ADC_BATTERY_CHANNEL);
        if (samples[i] > max_val) max_val = samples[i];
        if (samples[i] < min_val) min_val = samples[i];
        delay_ms(5);  // 采样间隔，避免连续采样
    }
    
    // 去掉最大最小值后求平均
    for (int i = 0; i < ADC_SAMPLE_COUNT; i++) {
        if (samples[i] != max_val && samples[i] != min_val) {
            sum += samples[i];
        }
    }
    
    uint16_t avg_adc = sum / (ADC_SAMPLE_COUNT - ADC_DISCARD_COUNT);
    float voltage = (avg_adc * 3.3f / 4095.0f) * 2.0f;  // 分压比1/2
    return voltage;
}
```

**精度指标**:
| 参数 | 数值 | 说明 |
|------|------|------|
| 测量范围 | 3.0V ~ 4.2V | 电池工作电压 |
| 分辨率 | ~2mV | 12bit ADC |
| 测量误差 | <±50mV | 含硬件误差和软件滤波 |
| 更新频率 | 1Hz | 每秒更新一次 |

### 10.2 分区设计要点

| 区域 | 设计要点 |
|------|----------|
| 电源区 | 大电流走线加粗，TP4056散热考虑 |
| MCU区 | 晶振靠近芯片，去耦电容靠近电源脚 |
| DFPlayer区 | 模块靠近扬声器接口，USB接口对齐外壳开口 |
| 传感器区 | HC-SR501可外接延长线，避开热源 |
| 接口区 | 按键、LED、充电口布局合理，便于操作 |

### 10.3 关键走线要求

**电源走线**:
- 5V主电源：线宽 ≥ 1mm（DFPlayer播放时电流较大）
- 3.3V电源：线宽 ≥ 0.5mm
- 充电回路：线宽 ≥ 1.5mm
- DFPlayer供电：建议在模块入口处加100uF滤波电容

**信号走线**:
- UART信号：PA9/PA10直连DFPlayer，线长尽量短（<10cm）
- UART串联电阻：建议100Ω串接在TX/RX线上抑制反射
- 红外信号：远离5V电源线，避免干扰
- BUSY信号：上拉输入，检测播放状态

### 10.4 安装孔位

### 12.3 常见问题排查

| 问题现象 | 可能原因 | 排查方法 |
|----------|----------|----------|
| 无法开机 | 电池无电/接触不良 | 测电池电压，检查接触 |
| MCU不工作 | 晶振未起振/程序未烧 | 检查晶振波形，重新烧录 |
| 红外不触发 | 灵敏度太低/接线错误 | 调节电位器，检查接线 |
| DFPlayer无响应 | 供电不足/U盘问题 | 检查5V供电，更换U盘测试 |
| DFPlayer不播放 | U盘格式/文件命名 | 确认FAT32格式，文件命名001.mp3等 |
| 串口通信失败 | 波特率/接线错误 | 确认9600bps，检查TX/RX是否接反 |
| BUSY信号异常 | 上拉电阻/接线 | 检查PB12上拉，确认BUSY接线 |
| 电流过大 | 短路/元件损坏 | 分段断电排查 |

### 12.4 DFPlayer调试技巧

**独立测试DFPlayer**：
1. U盘插入DFPlayer（预置001.mp3文件）
2. 上电（5V），DFPlayer应自动播放第一首
3. 用万用表测量BUSY脚：播放时为低电平(~0V)，空闲时为高电平(~3.3V)
4. 若能自动播放，说明模块和U盘工作正常

**串口指令测试**：
```
# 播放第1首
发送: 7E FF 06 0F 00 00 01 FE E5 EF

# 设置音量20 (0-30)
发送: 7E FF 06 06 00 00 14 FE D6 EF

# 暂停
发送: 7E FF 06 0E 00 00 00 FE ED EF
```
